[인터페이스] Interface
	- 정의 : 서로 다른 시스템이나 장치 사이에서 정보를 주소 받는 접점이나 경계면
	
	- 예시 : 키보드, 핸드폰, 키오스크, 리모컨 등등
		1. 삼성TV, LGTV 등등 서로 다른 회사의 TV들을 하나의 리모컨으로 조작한다.
		2. 오락실 : 서로 다른 여러 게임들을 하나의 게임기 기계로 조작한다.
		3. 컴퓨터 : 서로 다른 프로그램/게임들을 하나의 키보드에서 조작한다.
			- 사냥게임에서는 키보드의 A키를 누르면 공격
			- 축구게임에서는 키보드의 A키를 누르면 슛
			
	- 목적 : 서로 다른 장치들 사이에 동일한 목적으로 기능을 서로 다르게 작동하기 위해서 사용한다.
		- 자바 : 서로 다른 클래스들에서 동일한 목적으로 서로 다른 메소드를 실행한다.
		
	- 다형성 : 다양한 형식(형태, 모양, 타입)이 갖는 성질
		- 자바 : 개발코드와 객체가 서로 통신하는 접점 역할
		
	- 특징
		1. 객체의 교환성을 높이기 위해 다형성으로 구현
		2. 서로 다른 클래스들사이에 중간 매개역할
		
	- 인터페이스 작성 방법
		1. 패키지 오른쪽 클릭 --> [NEW] --> interface
		2. interface의 이름 작성 : 첫글자를 대문자로 작성
		
	- 인터페이스 구성 멤버
		1. 상수
			- public static final 타입 상수명 = 초기값;
			- public static final은 생략이 가능하다
			- 초기값은 필수이다.
		2. 추상메소드
			- abstract 생략 가능하다.
			- { }를 작성하지 않는다.
				- 추상메소드는 메소드의 구체적인 기능을 선언하지 않는다.
		*추상 : 여러가지 사물이나 개념에서 공통되는 특성이나 속성따위를 추출 파악
